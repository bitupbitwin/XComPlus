# X-COM+ 串口调试工具 开发说明书

复现 ALIENTEK XCOM 串口调试助手的核心功能，自用工具，只做与串口收发直接相关的功能。

## 1. 技术选型

| 项 | 选择 | 理由 |
|---|---|---|
| 语言 | Python 3.8+ | 开发快，跨平台 |
| GUI | PySide6 (Qt6) | 布局能力强，贴近原版界面 |
| 串口 | pyserial | 事实标准库 |

## 2. 功能范围

### 2.1 串口参数设置（右侧面板）
- 串口选择：自动枚举系统串口，提供刷新按钮
- 波特率：1200 ~ 921600 常用档位，可手动输入自定义值
- 数据位：5 / 6 / 7 / 8
- 停止位：1 / 1.5 / 2
- 校验位：无 / 奇 / 偶 / Mark / Space
- 流控：无 / RTS-CTS / XON-XOFF
- 打开 / 关闭串口（占用、拔出等异常要有提示并恢复状态）
- DTR / RTS 电平勾选控制

### 2.2 接收区
- 后台线程读串口，主线程刷新显示，不卡 UI
- 16 进制显示切换（切换时已收数据重新渲染）
- 时间戳显示：每包数据前加 `[HH:MM:SS.mmm]`
- 停止显示（暂停刷新但继续计数）
- 清除接收
- 保存窗口：把接收区内容存为 txt
- 字符编码：UTF-8 / GBK 可选
- 接收缓冲设上限（1 MB），超出丢弃最旧数据，防止长时间运行内存涨爆

### 2.3 发送区
- 单条发送
- 16 进制发送（"AA BB 0F" 格式解析，非法字符提示）
- 发送新行（自动附加 \r\n）
- 定时发送：周期 ms 可设
- 发送文件：选文件后按原始字节发出
- 清除发送

### 2.4 多条发送页签
- 按标签分组：可添加/删除标签（如 OS、AF），每个标签是独立一页
- 每页 20 条命令，左右两列各 10 条；每条：HEX 勾选 + 内容输入 + 独立发送按钮
- 循环发送：对当前标签页内非空条目顺序轮发，周期可设；切换标签自动停止
- 持久化：所有标签及命令内容自动保存到 `xcom_multisend.json`，重启不丢

### 2.5 状态栏
- 串口当前状态（已打开的端口@波特率 / 已关闭）
- RX / TX 字节计数 + 复位计数按钮

### 2.6 不做的内容
协议传输页、帮助页、关于/版权声明、皮肤、升级检查等一律不做。

## 3. 代码结构

```
X-com-plus/
├── 开发说明书.md
├── requirements.txt
├── run.py                  # 入口
└── xcom/
    ├── __init__.py
    ├── serial_manager.py   # 串口打开/关闭 + 后台收数线程(QThread)
    ├── multi_send.py       # 多条发送页签
    └── main_window.py      # 主窗口、收发页签、状态栏
```

## 4. 关键设计

- **收数线程**：`SerialWorker(QThread)` 循环 `read(in_waiting or 1)`（50ms 超时），
  收到数据通过 Qt 信号发给主线程；串口异常（如设备拔出）发 error 信号，主线程关串口并提示。
- **显示模型**：接收数据按"包"存为 `(时间, bytes)` 列表，HEX/时间戳开关切换时
  用该列表整体重渲染，保证切换前后数据一致。
- **HEX 解析**：去空格后必须是偶数个 16 进制字符，否则弹框提示不发送。
- **定时/循环发送**：用 `QTimer`，串口关闭时自动停止。

## 5. 运行

```bash
pip install -r requirements.txt
python run.py
```
