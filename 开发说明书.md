# X-COM+ 串口调试工具 开发说明书

复现 ALIENTEK XCOM 串口调试助手的核心功能，自用工具，只做与串口收发直接相关的功能。

## 1. 技术选型

| 项 | 选择 | 理由 |
|---|---|---|
| 语言 | Python 3.8+ | 开发快，跨平台 |
| GUI | PySide6 (Qt6) | 布局能力强，贴近原版界面 |
| 串口 | pyserial | 事实标准库 |

## 2. 功能范围

### 2.1 串口参数设置（右侧面板）
- 串口选择：点击下拉框时自动枚举系统串口（无需刷新按钮）
- 波特率：1200 ~ 921600 常用档位，可手动输入自定义值
- 数据位：5 / 6 / 7 / 8
- 停止位：1 / 1.5 / 2
- 校验位：无 / 奇 / 偶 / Mark / Space
- 流控：无 / RTS-CTS / XON-XOFF
- 打开 / 关闭串口（占用、拔出等异常要有提示并恢复状态）
- DTR / RTS 电平勾选控制

### 2.2 接收区
- 后台线程读串口，主线程刷新显示，不卡 UI
- 16 进制显示切换（切换时已收数据重新渲染）
- 时间戳显示：每包数据前加 `[HH:MM:SS.mmm]`
- 停止显示（暂停刷新但继续计数）
- 清除接收
- 保存窗口：把接收区内容存为 txt
- 字符编码：UTF-8 / GBK 可选；增量解码，多字节字符跨包接收不乱码
- 接收缓冲设上限（1 MB），超出丢弃最旧数据；显示区限 5 万行 / 8M 字符，
  超限按缓冲重渲染，防止长时间运行内存涨爆

### 2.3 发送区
- 发送区压缩为两行（输入框+发送按钮 / 选项行），日志接收区占据主要界面空间
- 单条发送
- 16 进制发送（"AA BB 0F" 格式解析，非法字符提示）
- 发送新行（自动附加 \r\n）
- 定时发送：周期 ms 可设
- 文件发送：打开文件（选择并显示路径）、发送文件（流式分块发出）、
  停止发送（中止文件发送与定时发送）、进度条实时显示百分比
- 清除发送

### 2.4 多条发送
- 与单条发送同在主界面下方的页签里，接收日志区始终在上方可见，
  点多条命令时 log 实时滚动；中间分割条可拖动调整比例
- 按标签分组：标签名完全自定义，标签行末尾固定 "+" 号点击即添加；
  右键标签弹出 重命名 / 删除（删除该标签及其全部命令，需确认）
- 标签内分页：每页 10 条命令，左右两列各 5 条；命令编号随页全局递增
  （第 2 页为 11~20）。页操作栏：页码 x/y、移除此页、添加页、
  首页/上一页/下一页/尾页、页码跳转
- 每条命令：HEX 勾选 + 内容输入 + 独立发送按钮
- 发送新行选项：勾选后每条命令自动附加 \r\n（默认开启）
- 循环发送：对当前标签当前页内非空条目顺序轮发，周期可设；切换标签自动停止
- 编辑条目：页操作栏「编辑条目」弹出表格对话框，批量编辑当前标签全部命令
  （HEX/指令/描述三列），支持清空、导入/导出 CSV（Excel 可直接打开）
- 持久化：所有标签及命令内容自动保存到 `xcom_multisend.json`，重启不丢

### 2.5 状态栏
- 串口当前状态（已打开的端口@波特率 / 已关闭）
- RX / TX 字节计数 + 复位计数按钮
- 当前时间（每秒刷新）

### 2.6 不做的内容
协议传输页、帮助页、关于/版权声明、皮肤、升级检查等一律不做。


## 4. 关键设计

- **收数线程**：`SerialWorker(QThread)` 循环 `read(in_waiting or 1)`（50ms 超时），
  收到数据通过 Qt 信号发给主线程；串口异常（如设备拔出）发 error 信号，主线程关串口并提示。
- **显示模型**：接收数据按"包"存为 `(时间, bytes)` 列表，HEX/时间戳开关切换时
  用该列表整体重渲染，保证切换前后数据一致。
- **HEX 解析**：去空格后必须是偶数个 16 进制字符，否则弹框提示不发送。
- **定时/循环发送**：用 `QTimer`，串口关闭时自动停止；tick 内有重入保护，
  避免发送失败弹出的模态对话框期间定时器再触发导致对话框叠加。
- **写超时**：串口 write 设 2s 超时，流控（CTS）阻塞时提示发送超时，
  不会卡死界面。

## 5. 运行

```bash
pip install -r requirements.txt
python run.py
```
